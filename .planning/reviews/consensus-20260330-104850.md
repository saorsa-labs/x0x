## MUST FIX Items (2+ votes)

### 1. NodeCreation wrong error variant (4 votes)
**Finding**: `error::NetworkError::NodeCreation` used for "presence not initialized" — semantic mismatch.
`NodeCreation` implies failure during node construction, not an API precondition.
**Fix**: Change to `error::NetworkError::NodeError("presence system not initialized".to_string())`
in `subscribe_presence()`, `discover_agents_foaf()`.
**Files**: `src/lib.rs`

### 2. machine_public_key empty in fallback DiscoveredAgent (3 votes)  
**Finding**: When creating fallback `DiscoveredAgent` (peer not yet in identity cache),
`machine_public_key: Vec::new()` is used. Downstream code that tries to verify a rendezvous
`ProviderSummary` using this key will fail silently or panic.
**Fix**: Document the limitation clearly in the fallback path. Add a note that callers of
`presence_record_to_discovered_agent` should treat entries with empty `machine_public_key`
as unverified until enriched by identity heartbeat. Alternatively, use a sentinel constant.
**Files**: `src/presence.rs`

### 3. O(n) linear scan in peer_to_agent_id (4 votes)
**Finding**: `peer_to_agent_id` iterates all cached agents to find one by machine_id.
At scale (thousands of agents) this is O(n) per event-loop poll cycle.
**Fix**: The fix needs a reverse index, but that requires changes to the cache structure
which is shared across the codebase. For Phase 1.2 scope, add a doc comment documenting
the O(n) complexity and acceptable scale limit (<10K agents). Full reverse index is Phase 1.4.
**Files**: `src/presence.rs`

## SHOULD FIX Items (2 votes)

### 4. No unit tests for presence helpers (2 votes)
**Finding**: `peer_to_agent_id`, `presence_record_to_discovered_agent`, `parse_addr_hints`
have no unit tests.
**Fix**: Add `#[cfg(test)]` module to `src/presence.rs` with unit tests.
**Files**: `src/presence.rs`

### 5. Silent broadcast send failure (2 votes)
**Finding**: `let _ = event_tx.send(...)` silently discards `SendError` when channel has no receivers.
**Fix**: Add `tracing::debug!` when send fails (no receivers is normal, but worth noting).
**Files**: `src/presence.rs`
